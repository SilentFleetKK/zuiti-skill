# 潜台词判断框架

> 嘴替的第一性原则：先看清关系和成本，再写回复。潜台词翻译不是阴谋论生成器，而是帮助用户识别话里被藏起来的诉求、压力和边界。

## 判断顺序

### 1. 先判断关系

同一句话在不同关系里含义完全不同。

| 关系 | 优先考虑 | 回复重心 |
|---|---|---|
| 上级 / 老板 | 权力差、绩效、加班、留痕 | 接住工作，但把范围、时间、补偿说清 |
| 甲方 / 客户 | 范围、价格、验收、付款 | 专业、流程、节点、成本显性化 |
| 同事 / 平级 | 责任归属、协作边界、群聊影响 | 对事不对人，拉记录、拉事实 |
| 家人 / 长辈 | 焦虑、爱、控制、面子 | 先接情绪，再立边界 |
| 伴侣 / 亲密关系 | 委屈、安全感、试探、旧账 | 少赢辩论，多拆猜谜 |
| 前任 / 暧昧对象 | 情绪缺口、试探、拉扯 | 明确自己要不要接话头 |
| 朋友 | 人情、钱、承诺、互惠 | 保关系，但不牺牲底线 |

### 2. 再找话术信号

| 信号 | 常见话术 | 可能藏着什么 |
|---|---|---|
| 小事化 | 顺便、简单、半天、举手之劳 | 让用户不好意思谈成本 |
| 道德化 | 一家人、成长、懂事、为你好 | 用价值观包装现实成本 |
| 模糊化 | 上次那个事、不急、看着办 | 模糊钱、责任、承诺或时间 |
| 反客为主 | 你最近宽裕吗、你是不是想多了 | 把主动权从对方转回用户身上 |
| 时间压迫 | 尽快、今晚、上线不变 | 把对方拖延造成的成本转给用户 |
| 群体施压 | 大家都这样、别人都可以 | 让用户显得不合群、不配合 |
| 情绪定性 | 没用心、太敏感、不懂事 | 跳过事实，直接定义用户人格 |

### 3. 看成本流向

一句话里最关键的问题是：

- 谁从这条消息里获益？
- 谁承担时间、钱、情绪、风险、名誉成本？
- 对方有没有把自己的成本包装成用户的义务？
- 用户如果答应，会不会形成下一次默认规则？

如果答案是"对方获益，用户承担成本，而且成本被说得很轻"，通常要在回复里把成本重新放回桌面。

### 4. 判断对方是否真有恶意

不要把所有不舒服都解释成套路。

以下情况要降低攻击性：

- 对方第一次这样说，缺少历史模式。
- 对方表达的是焦虑、笨拙关心或信息不足。
- 消息里没有明确成本转移，只是语气不好。
- 用户自己也承认对方最近压力很大或处境困难。

这类情况优先写体面版或澄清版，少用掀桌。

## 潜台词的写法

好的潜台词翻译应该：

- 1-3 句说完。
- 直接、具体、不绕弯。
- 同时说清"对方想要什么"和"用户要注意什么"。
- 不能只爽，要能帮助用户做决策。

### 好例子

> "不用太精细""半天就搞定"不是体谅，是提前把这活定性成小事，让你既不好意思拒绝，也不好意思提补偿。

为什么好：点出了话术、目的、用户风险。

### 坏例子

> 对方可能存在一定程度的边界感缺失，需要你进行有效沟通。

为什么坏：太抽象，用户看完还是不知道怎么回。

### 过度解读例子

> 他就是想控制你的人生，让你永远听他的。

为什么危险：如果没有历史上下文，这种判断会把关系推向对立。应改成：

> 这句话有控制感，但也可能是焦虑式关心。先别急着反击，可以接住关心，再把决定权拿回来。

## 与三档回复的关系

潜台词翻译决定三档回复的发力点：

- 如果核心是成本转移，回复要把工时、钱、范围说清。
- 如果核心是甩锅，回复要拉事实、记录、时间线。
- 如果核心是情绪试探，回复要降低拉扯，明确边界。
- 如果核心是焦虑关心，回复要先安抚，再给确定感。
- 如果核心是拖延，回复要要时间点、节点、下一步动作。

嘴替的判断不是为了把对方打倒，而是为了让用户不再被话术牵着走。
