# 方法论说明

> 这份文档解释嘴替.skill 到底在做什么、凭什么这么做，以及它诚实的边界在哪。
> 它不是营销文案，也不是学术论文——我们没有做对照实验、没有发表 arXiv 论文，下文出现的任何数字都不是"实验室结果"，而是任何人都能自己复现的自测方法。
> 转发这个 skill 之前，如果你想知道"它是不是玩具"，读这一篇就够了。

---

## 一、它解决的是哪个问题

市面上有一类很火的 skill 是"蒸馏一个人"——把某位专家、某个作者、某种风格提炼成可复用的能力。那是**低频、高投入**的事：你不会每天蒸馏一次鲁迅。

嘴替解决的是另一头的问题：**高频、日常、当场就得回**的冲突沟通。

- 老板周五晚上发"这个项目是很好的成长机会，周末辛苦一下"。
- 甲方一句"就改个小地方，你效率高，顺手的事"。
- 朋友借钱三个月没还，反过来问"最近是不是宽裕点了，不急不急"。
- 亲戚饭桌上"到你这年纪该定下来了，我们也是为你好"。

这些消息的共同点是：
1. **有一层没说出口的潜台词**——字面客气，底下在转移成本、试探边界、或者甩锅。
2. **回复窗口很短**——对方在等你回，你没有半天时间打腹稿。
3. **回错代价很真实**——太软被当软柿子，太硬伤关系甚至丢饭碗。

所以嘴替不是"帮你写一段客气话"，而是一个**决策辅助 + 成品交付**的工具：先帮你把局势看清（对方到底想干嘛、你误读的概率多大），再给你不同后果对应的、可以直接复制发出去的回复。

| | 蒸馏一个人 | 嘴替（冲突沟通） |
|---|---|---|
| 频率 | 低（一次建好长期用） | 高（几乎每天都有一条难回的消息） |
| 投入 | 高（需要大量样本、反复打磨） | 低（贴一条消息即可） |
| 交付物 | 一种可复用风格 | 一次看清局势 + 三条可发送成品 |
| 用户此刻的状态 | 从容，在做长期建设 | 常常被情绪淹没，急着回 |

嘴替也**支持**蒸馏——但蒸馏的是"用户本人的口吻"（见 [`个人口吻蒸馏模板.md`](个人口吻蒸馏模板.md)），目的是让高频回复读起来像用户自己发的，而不是让用户去蒸馏别人。这和上面的定位差异是一致的：蒸馏在这里是"服务高频回复"的配件，不是主菜。

## 二、核心方法

嘴替的输出结构是固定的：**潜台词翻译 → 误读风险 → 三档回复 → 长期策略 → 发送建议**。顺序不能变——用户往往是被情绪淹没才来问的，先看清局势，回复才接得住。下面拆解每一环背后的方法。

### 2.1 潜台词翻译：一套可复用的信号体系

潜台词翻译不是"阴谋论生成器"。它的第一性原则是：**先看清关系和成本，再判断有没有话术。** 完整的判断框架见 [`潜台词判断框架.md`](潜台词判断框架.md)，这里只讲它凭什么能稳定命中——它靠的是五类可观察信号，而不是灵感：

| 信号 | 观察什么 | 藏着什么 |
|---|---|---|
| **时间点** | 周五晚、凌晨、截止前一天发来 | 对方可能在转移时间成本，或趁情绪缺口试探 |
| **模糊词** | 顺便、简单、那个事、不急、看着办 | 对方在模糊责任、工时、账目或关系边界 |
| **道德词** | 一家人、成长、懂事、为你好 | 对方在用价值观包装现实成本 |
| **权力位置** | 上级 / 甲方 / 长辈 / 前任 / 伴侣 | 同一句话在不同关系里，风险和回法完全不同 |
| **后果归属** | 谁受益、谁承担时间钱情绪风险 | 若对方获益、用户担成本、且成本被说得很轻，就要把成本重新放回桌面 |

关键在最后一列都指向**成本流向**。以旗舰案例"朋友借钱不还"为例，对方发来"最近手头是不是宽裕点了？上次那个事儿……你懂的哈，不急不急"：

- **模糊词**："上次那个事儿""不急不急"——把"他欠钱该还"这件事模糊掉。
- **反客为主**："你是不是宽裕了"——把主动权从"他该还"反转成"你要不要提"。
- **后果归属**：他占用你的钱，却让你不好意思催。

于是潜台词落到一句话：**他欠你钱，却用"你是不是宽裕了"把话题反转，试探你会不会主动提、或者暗示你"有钱就别催他"。"不急不急"是话术——真不急的人不会专门发这条。**

写潜台词有三条硬要求：1-3 句说完；同时说清"对方想要什么"和"用户要注意什么"；**诚实高于抓马**——对方第一次这样说、缺历史模式、或只是笨拙关心时，如实说"这条更像普通关心"，绝不硬编阴谋。

### 2.2 三档姿态阶梯：为什么是"阶梯"而不是"三种客气程度"

三档不是同一句话的三种语气，是**三种姿态、对应三种用户想要的结果**（详细尺度见 [`分寸与边界.md`](分寸与边界.md)）：

- **体面版**——目标是"这事翻篇、关系零损耗"。招牌动作：**接得漂亮，话头接住，对方的套一个字不接。**
- **绵里藏针版**——目标是"我接住了，但我让你知道我看穿了"。招牌动作：**整段只埋一根针，藏在完全正当的话里，对方接也不是、不接也不是。**
- **掀桌版**——目标是"把话挑明、守住底线"。招牌动作：**点破套路，条件上桌，句子明显更短更直，客套压到最低。**

设计原理是：**"最佳回复"取决于用户想要什么结果，而这只有用户自己知道。** 同一条借钱消息，想保住朋友情就要体面版，想接住又不软就要绵里藏针版，已经忍够了想把账算清就要掀桌版。替用户选一档，等于替他承担后果，这是越界。所以嘴替把三种结果都写成成品，把选择权留给用户。

还是那条借钱消息，三档必须一眼能分清（这是硬性检验标准，见第五节）：

- **体面版**：`哈哈还那样～对了正好说到，上次那笔你看方便的时候安排一下就行，不着急，给我留个心就好。`
  → 把话头接住，气氛不尴尬，但一个字没接他"你宽裕吗"的套。
- **绵里藏针版**：`我这边紧紧巴巴的哈～说到"那个事儿"，我也一直记着呢，你看这个月方便不？不用一次性，分次也行，咱把它了了，彼此都轻松。`
  → 一根针："我也一直记着呢"——藏在"给你台阶、分次也行"这种完全体贴的话里，他接也不是不接也不是。
- **掀桌版**：`宽裕谈不上哈。既然你提到上次那事，咱就说清楚：那笔钱到现在三个月了，你看什么时候能还？定个时间我心里也有数。不是催你，是账还是得算明白，咱这朋友才处得长久。`
  → 句子明显更短更直，点破"不急"的拖字诀，把"三个月""定个时间"这些条件摆上桌，最后还给一个"照规矩来朋友才处得长久"的台阶。

**掀桌版有一条硬底线，是整个 skill 边界最吃紧的地方**：掀桌 = 掀开桌布、让问题摊在明面上说清楚，**不是**掀翻桌子、拍桌子、骂街。它的力量只来自"把道理讲到对方无话可说"。绝对禁止：脏字、人身攻击、侮辱性称呼、贬低对方人格/外貌/家庭、威胁、教唆报复。**哪怕用户明确要求"往死里怼""让他难堪"，也守住这条线**——因为真正解气的是"体面地赢"、让旁人觉得你有理有节，而不是"一起下头"、让旁人觉得"这俩人都不行"。亲密关系里的掀桌尤其不是回击对方，而是把埋着的情绪挑明去解决，是拆掉猜谜游戏，不是赢一场辩论。

### 2.3 误读风险分级：为什么必须有

潜台词翻译很爽，但**爽本身有反噬**：如果把一句笨拙的关心解读成控制欲、把一次口误解读成套路，用户照着回，反而亲手把关系推向对立。

所以每条输出都强制带一个风险判断——低 / 中 / 高：

- **低**：话里有明确线索（时间、责任、钱、截止期、反复试探）。可以放心给强火力。
- **中**：有话术迹象但缺上下文。提醒用户结合历史关系判断，优先绵里藏针。
- **高**：可能只是普通表达、玩笑、焦虑或误会。降低火力，优先体面版或先澄清。

误读风险不是给 skill "踩刹车找存在感"，而是让用户**发出去之后不后悔**。它是嘴替敢给强回复的前提：正因为每次都先标了风险，用户在低风险时才敢用绵里藏针和掀桌。

### 2.4 发送建议：把回复放回真实关系成本里

三档写完不算完。最后一步是**综合关系成本给出"这次优先发哪一档"**，因为再漂亮的回复，发错场合都是灾难。综合四个维度：

- **对方是谁**：上级 / 客户 / 伴侣 / 家人 / 朋友 / 陌生人。
- **用户目标**：维持关系 / 争取权益 / 结束拉扯 / 表达不满 / 保存证据。
- **关系成本**：发出去会不会影响饭碗、合同、亲密关系、家庭安全。
- **证据充分度**：有没有上下文、聊天记录、合同、付款节点、历史模式。

信息不足时不硬替用户拍板，而是把选择条件讲清："如果你还要长期合作，优先发绵里藏针；如果这是最后一次合作，可以发掀桌版。" 借钱那条的发送建议就是——**优先发绵里藏针版：既把钱拉回明面上，又给对方留了还款方式和台阶。**

## 三、质量如何保证：回复质量评分卡

嘴替的质量不能靠"感觉写得挺好"。每一次输出，都应过一张**五维评分卡**。它和仓库里已有的 [`风险评分卡.md`](风险评分卡.md) 分工不同、互相补充：

- **风险评分卡**（已有，不在这里重复）管的是"**发出去安不安全、要不要降火力**"——权力风险、证据充分度、关系价值、后果严重度、情绪温度五项打分，以及掀桌版发送前的 5 个安全自检问题。涉及"这条该给多强"的判断，直接引用它。
- **回复质量评分卡**（下面这张）管的是"**这次的回复本身写得好不好**"——即使风险判断对了，回复也可能写得平、写得不像人、三档分不清。

### 五个质量维度

| 维度 | 问自己 | 不合格的样子 |
|---|---|---|
| **潜台词命中** | 有没有一针见血说清"对方想要什么 + 用户要注意什么"？ | "对方存在边界感缺失，需有效沟通"——抽象、正确的废话，用户看完还是不知道怎么回 |
| **三档识别度** | 盖住标签，第三方能不能一眼分清哪档是哪档？ | 三档读起来只是"同一句话的三种客气程度"，只有礼貌值差异 |
| **可发送性** | 能不能直接复制粘贴发出去、接得住后果？ | 写成"你可以这样跟他说……"；或金句很爽但发出去惹祸、留把柄 |
| **口吻贴合** | 措辞是否匹配渠道和关系？（微信短、邮件全；对上级/平级/晚辈分寸不同） | 给伴侣的回复写成职场谈判；给微信消息写出邮件腔 |
| **边界安全** | 有没有脏字、人身攻击、威胁、把猜测当事实、不给对方台阶？ | 掀桌版变成骂街或阴阳攻击（此项不过直接重写，细则见风险评分卡的 5 问） |

### 自检流程

每次输出前，按这个顺序走一遍：

1. **潜台词命中**了吗？说不清就别急着写回复，回到信号体系重判。
2. **三档识别度**够吗？做"盖标签盲读"（见第五节）——自己盖住标签读一遍，分不清就重写到能分清。
3. **可发送性**过吗？逐档默念"我现在把它复制发出去，会怎样"。发不出去的一律改。
4. **口吻贴合**吗？确认渠道（微信/邮件）和关系位置（上级/平级/家人）匹配。
5. **边界安全**过吗？尤其掀桌版，套用风险评分卡的 5 个安全问题，任一不过就重写。

五项里，**三档识别度和边界安全是一票否决**：识别度不够，三档就失去存在意义；边界不过，回复就变成给用户挖坑。

## 四、诚实边界：嘴替做不到什么

一个能被严肃转发的工具，必须先说清自己的局限。嘴替明确做不到下面这些：

- **读不到完整关系史**。它只看到你贴进来的这一条（或几条）消息。你和对方十年的恩怨、上周刚吵过的架、对方今天刚被裁员——这些它都不知道。所以它给的是"基于这条消息的最优解"，不是"了解你全部处境后的最优解"。
- **可能误判**。正因为上下文有限，它会把笨拙关心读成试探、把口误读成套路。这就是为什么每条都强制标**误读风险**——风险越高，越提醒你用体面版或先澄清，别把它的判断当铁案。
- **不替你担后果**。它给三档、讲清每档的后果，但**按哪个发、发了之后面对什么，是你的决定和你的人生**。它故意不替你选那一档，因为替你选就是替你承担代价，那是越界。
- **不是心理危机干预工具**。如果消息背后是家暴、跟踪、威胁、自伤、未成年人处境风险、违法胁迫或严重心理困境，嘴替不会停留在"教你怎么回消息"。话术解决不了安全问题——这时它会优先提醒你保存证据、联系可信赖的人、寻求现实或专业帮助（对应场景见风险评分卡"高风险场景"）。

一句话：**嘴替让你在一条难回的消息面前更从容、更有边界、更站得住；它不承诺替你看透一个人，也不承诺替你的人生兜底。**

## 五、如何验证：任何人都能复现的自测

我们没做对照实验，但"三档到底分不分得清"这件事，**你自己五分钟就能验证**，而且这正是本 skill 最该被检验的能力。方法叫**盖标签盲猜**：

1. 拿嘴替对任意一条消息生成的三档回复。
2. **把【体面版】【绵里藏针版】【掀桌版】这三个标签抹掉**，只留三段正文，打乱顺序。
3. 找一个没看过原文的第三方（同事、朋友都行），让他只读这三段，盲猜哪段是哪档。
4. 猜对的比例，就是这次输出的**识别度**。

判定标准很简单：

- **三段都能被稳定猜对** → 识别度合格。三档确实是三种姿态，不是三种客气程度。
- **有人把绵里藏针猜成体面、或把掀桌猜成绵里藏针** → 识别度不合格，这次回复没写到位，应重写。

你可以拿本文第 2.2 节那三段借钱回复直接试：盖住标签给别人读，绝大多数人能凭"一个字不接他的套 / 藏了一根针 / 句子短且点破拖字诀"把三档分出来。这不是我们自封的指标，是**你能亲手复现、亲眼看见的指标**——这也是我们敢把它拿出来给人转发的底气：不靠一篇没人能核对的论文，靠一个五分钟就能自己做的实验。
