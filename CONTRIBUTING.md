# Contributing

欢迎给嘴替.skill 贡献案例、文档、传播素材、专属口吻模板和运行改进。

这个项目最需要的不是抽象功能点，而是真实、难回、有分寸的沟通案例。

## 双轨制：核心护方法论，社区走索引

**SKILL.md 是本仓库的核心资产，外部 PR 不直接往里合。** 嘴替的潜台词框架、三档定义、安全边界都经过维护者实测校准；发现方法论 bug 或改进点，请开 Issue 聊——采纳了会由维护者写进 SKILL.md，并在提交里给你署名致谢。

**你蒸馏的"专属嘴替"和自制场景包单独成仓，不进本仓库的 personas/。** `personas/template` 和 `examples/` 是维护者出品的官方口径。你的作品这样走：

1. 放进你自己的 GitHub 仓库，一个嘴替/一个场景包一个仓库，**star 和维护权全归你自己**；
2. 开个 PR，把仓库链接登记到 [COMMUNITY.md](COMMUNITY.md) 的索引表，一行就行。

这样你的作品有自己的门牌、自己的 star，迭代节奏也不受本仓库牵制。

## 可以贡献什么

### 1. 难回消息案例

优先级最高。适合投稿的场景：

- 老板画饼、临时派活、甩锅、PUA。
- 甲方压价、拖款、加需求、反复横跳。
- 亲戚催婚、比较、越界关心。
- 伴侣阴阳怪气、前任试探、朋友借钱不还。
- 拒绝别人、谈钱、道歉、追责、谈薪。

请务必脱敏：

- 删除姓名、公司名、手机号、地址、订单号、合同号。
- 不提交完整聊天记录，只提交必要上下文。
- 不提交会暴露真实身份的截图原图，建议转成文字。

### 2. 三档回复改进

好的三档回复应该：

- 体面版不卑微。
- 绵里藏针版只藏一根针。
- 掀桌版有理有据，不脏、不威胁、不攻击人格。
- 三档一眼能分出来，不是同一句话换语气。

### 3. 风险与边界补充

如果你发现某类场景容易误伤关系，欢迎补充到：

- `references/风险评分卡.md`
- `references/分寸与边界.md`
- `references/潜台词判断框架.md`

### 4. 传播素材

传播卡片由 `tools/generate_promo_assets.py` 生成。新增卡片时：

1. 在脚本的 `CARDS` 里添加案例。
2. 运行 `python3 tools/generate_promo_assets.py`。
3. 检查 `promo/cards/` 是否生成对应 SVG。

## 案例格式

```markdown
## 场景标题

**关系**：老板 / 甲方 / 亲戚 / 伴侣 / 朋友 / 其他

**上下文**：一句话说明，不要泄露隐私。

**对方消息**：
> 这里贴消息

**希望达成的结果**：
维持关系 / 争取权益 / 拒绝 / 要钱 / 结束拉扯 / 其他

**你希望的力度**：
体面 / 绵里藏针 / 掀桌 / 不确定
```

维护者会把合适案例整理进 `examples/` 或 `promo/cards/`。

## 不接受的内容

- 人身攻击、羞辱、威胁、骚扰、报复教程。
- 未脱敏的私人聊天记录。
- 鼓励违法、造谣或现实伤害的回复。
- 只图爽但发出去会让用户吃亏的表达。

嘴替追求的是"体面地赢"，不是一起失控。

## 本地检查

提交前建议运行：

```bash
python3 tools/generate_promo_assets.py
python3 - <<'PY'
from pathlib import Path
import xml.etree.ElementTree as ET
for p in [Path('assets/banner.svg'), *Path('promo/cards').glob('*.svg')]:
    ET.parse(p)
    print('ok', p)
PY
```

如果只是改 markdown，检查链接和标题即可。
